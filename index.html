C.

Issues with the current design:

• The buttons labelled ‘M’ and ‘S#’ may confuse users not aware of what they stand for.
• The *G button does not light up when pressed.
• Queueing up several floors while the elevator doors are closing can cancel the whole queue and turn off all the lights of the chosen floors.
• The interface does not indicate which direction the elevator is moving.
• There is no light indicator for the open and close door buttons to show that request has been acknowledged if the action is not immediate.
• When choosing floors that the elevator can’t process at the time, the button lights either turn off immediately or
flash twice then stay off. The difference in the meaning between those is unintelligible since the outcome is the
same with the elevator indicating it cannot process that request currently. This could also be confused with the elevator
not working correctly or has a malfunction.


Explain why it is good/bad:

• Good
o The interface has incorporated brail for the buttons, helping some disabled users.
o The buttons are clearly marked with their corresponding labels.

• Bad
o Buttons labels can be confusing.
o Not all button light indicators work properly.


B.

List your most common uses and other more rare uses:

• Common
o To go up or down a building quickly.
o To avoid using the stairs.

• Rare
o Transporting patients via gurney in a hospital.
o Unable to use the building’s stairs do to disability or handicap.


Does the interface make doing those common things easier?

• The interface makes common use more tedious than normal due to having to
interpret the interface upon initial use and spending extra time doing so.


What is the common sequence of actions?

• Interpreting/understanding the elevator’s interface.
• Deciding what floor of the building you need.
• Pressing the corresponding floor button on the interface.


How does the elevator support the user to figure out how to make it work?

• The buttons have floor labels that are a distinct different color compared to
the metal frame and are put to the side of each button to clearly indicate which button is for which floor.


How does the elevator provide feedback to the user?

• If queueing up more than one floor, the button lights stay on to indicate the floor request has been processed and will proceed to that floor.
• The buttons either turn off immediately, or flash then stay off, instead of staying on when pressed when choosing a floor
that is in the opposite direction the elevator is currently moving. This indicates it is unable to process that request
at that time due to the current direction it is moving. This is determined through repeated use of the interface to
confirm the buttons do work when processed correctly and are not malfunctioning.
• When a light turns off a while after having been chosen, it indicates the floor has been reached.


What are some common mistakes you can make with this current design?

• You could confuse the ground floor with being floor S1 since it is common to consider the 1st floor to be the ground floor.
• Try to pick a floor that is going in the opposite direction the elevator is currently moving in and be denied.
